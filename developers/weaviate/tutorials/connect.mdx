## 概述

在这里，您将学习如何连接到您的Weaviate实例，包括如何提供身份验证参数，如Weaviate API密钥和推理API密钥（例如Cohere、Hugging Face或OpenAI API密钥）。

## 先决条件

在进行本教程之前，您需要知道：
- 您的Weaviate实例的位置，
- Weaviate实例的**配置**，特别是：
    - 是否启用了**身份验证**，如果是，允许的身份验证方法。
    - 如果使用了使用外部推理API（例如OpenAI）的**模块**。

import EmbeddedInstantiationSimple from '/_includes/code/embedded.instantiate.simple.mdx';

### 对于WCS用户

您可以在Weaviate Cloud Services仪表板中找到每个实例的集群URL、认证状态和启用的模块。

### 对于Docker/Kubernetes部署

您可以在配置文件中找到相关的配置，比如Docker Compose中的`docker-compose.yml`或Helm chart中的`values.yaml`。

对于Docker实例，您的URL可能是`http://localhost:8080`，除非另有指定。

:::tip For Embedded Weaviate
If you are using Embedded Weaviate, you do not need to specify a URL at the client instantiation step, and most likely will not need any authentication.
<br/>
<p>
So, you can simply instantiate with the following.
</p>

<EmbeddedInstantiationSimple/>

For the default configuration, and more about the option in general, see the [Embedded Weaviate documentation](../installation/embedded.md).
:::

## 无需身份验证

This set up is not recommended in general as anyone can access your Weaviate instance as long as they have the URL.

## 启用身份验证

如果您启用了身份验证，您可以使用 API 密钥（推荐）或 OIDC 登录。
