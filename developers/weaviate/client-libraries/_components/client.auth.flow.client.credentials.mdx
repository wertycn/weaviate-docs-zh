此 OIDC 流程使用 `client secret` 来获取所需的身份验证令牌。

这种流程适用于应用程序与 Weaviate 之间的服务器间通信，不涉及最终用户，并对应用程序进行身份验证。这种身份验证流程通常被认为比资源所有者密码流程更安全：一个被泄露的客户端密码可以被简单地撤销，而一个被泄露的密码可能会产生超出受损身份验证范围的更大影响。

要对客户机密进行身份验证，大多数身份提供者需要指定一个*范围*。这个*范围*取决于身份提供者的配置，因此我们要求您参考身份提供者的文档。

大多数提供者在其响应中不包含刷新令牌，因此`client secret`保存在客户端中，在现有令牌过期时获取新的`access token`。
